### SEGMENT 4: The Coordination Insight (2 min)

**[07:00 - 07:05] [A-CAM: Interviewer, asking for the core insight.]**

**INTERVIEWER:**
"So self-supervised learning is learning from coordination?"

**[07:05 - 07:50] [B-CAM: LeCun, nodding vigorously.]**

**LECUN:**
"Yes. Let me explain. In supervised learning, you have:

- Input: an image


- Label: 'cat'


- Learning signal: minimize error between prediction and label



The label provides the learning signal. But in self-supervised learning, you have:

- Input: multiple views of the same data


- Task: predict one view from another


- Learning signal: consistency across views



The coordination of multiple views creates the learning signal. No external labels needed."

**[07:50 - 07:55] [A-CAM: Interviewer, asking for an example.]**

**INTERVIEWER:**
"Give us a concrete example."

**[07:55 - 08:30] [B-CAM: LeCun, explaining with video.]**

**LECUN:**
"Video. You see a ball rolling. Current frame shows the ball here. Next frame shows it there. You learn to predict: 'If the ball is here now, it will be there next.' The learning signal comes from temporal consistency—the ball's motion is predictable. You're coordinating current frame with next frame with physical dynamics. No labels, just coordination."

## ACT II: THE THREE-BODY ARCHITECTURE OF LEARNING

**Duration: 24 minutes**

### SEGMENT 5: The Three Coordinating Elements (7 min)

**[08:30 - 08:35] [A-CAM: Interviewer, transitioning to architecture.]**

**INTERVIEWER:**
"You've said that deep learning is fundamentally about coordination—not just feedforward processing. Walk us through the architecture. What are the three coordinating elements?"

**[08:35 - 08:50] [C-CAM: LeCun walks to whiteboard, picks up marker.]**

**LECUN:**
"Most people think neural networks are simple: data flows in, processing happens in hidden layers, output comes out. But that misses the coordination that makes learning possible. Let me show you the three-body architecture."

**[08:50 - 09:00] [WHITEBOARD CAM: LeCun draws three circles in a triangle formation.]**

**LECUN:**
"Three elements coordinate during learning. Each one essential. Together, they create intelligent representations."

**[09:00 - 09:05] [WHITEBOARD CAM: LeCun labels first circle: "DATA (Bottom-Up)"]**

**LECUN:**
"**Element 1: Data Representations—Bottom-Up.**"

**[09:05 - 10:00] [B-CAM: LeCun, explaining data representations.]**

**LECUN:**
"This is what the network learns from raw data. You start with pixels, audio waveforms, text. The network transforms this through layers, creating representations—features—at each layer."

**[09:25] [SCREEN: Animation showing a CNN processing an image: Layer 1 (edges), Layer 2 (textures), Layer 3 (parts), Layer 4 (objects).]**

**LECUN:**
"In vision:

- Layer 1: Edges, colors, textures


- Layer 2: Corners, contours, simple shapes


- Layer 3: Parts—eyes, wheels, windows


- Layer 4: Objects—faces, cars, buildings



The network learns WHAT to represent. These features aren't programmed—they emerge from data. This is bottom-up: raw data flowing up through layers, becoming increasingly abstract."

**[10:00 - 10:05] [A-CAM: Interviewer, clarifying.]**

**INTERVIEWER:**
"So the network discovers useful features automatically?"

**[10:05 - 10:15] [B-CAM: LeCun, nodding.]**

**LECUN:**
"Exactly. A good representation captures the underlying structure of the data. It makes the task easy. That's what learning is—finding representations that make problems solvable."

**[10:15 - 10:20] [WHITEBOARD CAM: LeCun labels second circle: "OBJECTIVES (Top-Down)"]**

**LECUN:**
"**Element 2: Task Objectives—Top-Down.**"

**[10:20 - 11:10] [B-CAM: LeCun, explaining objectives.]**

**LECUN:**
"This is what we're trying to achieve. The objective function—what we're optimizing for. In supervised learning, it's 'minimize classification error.' In self-supervised learning, it's 'maximize prediction accuracy.' The objective provides the learning signal."

**[10:40] [SCREEN: Different objectives illustrated: classification loss, prediction error, contrastive loss.]**

**LECUN:**
"The objective shapes which representations are learned. Same network architecture, different objectives → different representations. For image classification, you learn features that distinguish categories. For object detection, you learn features that localize objects. The objective is top-down pressure that guides learning."

**[11:10 - 11:15] [A-CAM: Interviewer, seeing the coordination emerging.]**

**INTERVIEWER:**
"So data provides raw material, objectives provide direction. What's the third element?"

**[11:15 - 11:20] [WHITEBOARD CAM: LeCun labels third circle: "ARCHITECTURE (Structure)" and connects all three with arrows.]**

**LECUN:**
"**Element 3: Architecture Priors—Structural.**"

**[11:20 - 12:30] [B-CAM: LeCun, explaining architectural priors.]**

**LECUN:**
"This is the structure of the network—the built-in assumptions about the world. Different architectures embody different priors:

**Convolutional networks:**

- Prior: Spatial locality (nearby pixels are related)


- Prior: Translation invariance (a cat is a cat anywhere in the image)


- Structure: Shared weights, local receptive fields



**Attention mechanisms:**

- Prior: Some parts of input are more relevant than others


- Structure: Learn to weight importance dynamically



**Recurrent networks:**

- Prior: Sequential dependencies matter


- Structure: Maintain hidden state over time"



**[12:00] [SCREEN: Three architectures illustrated with their key structural features.]**

**LECUN:**
"The architecture isn't just a technical detail—it encodes our assumptions about the problem. Good architectures make learning possible. Bad architectures make learning impossible. The architecture determines HOW to represent."

**[12:30 - 12:35] [WHITEBOARD CAM: LeCun draws arrows connecting all three elements in a coordination loop.]**

**LECUN:**
"Now here's the key: these three don't work in isolation. They coordinate."

### SEGMENT 6: How Coordination Creates Intelligence (6 min)

**[12:35 - 12:40] [A-CAM: Interviewer, asking about the coordination process.]**

**[12:40 - 13:20] [B-CAM: LeCun, explaining the coordination dance.]**

**LECUN:**

**[13:00] [SCREEN: Animation showing not just forward/backward passes, but three forces shaping representation space simultaneously.]**

**LECUN:**
"Three forces shape the representation space simultaneously:

1. **Architecture constrains** what representations are possible


2. **Data provides** raw material and structure


3. **Objective guides** which representations are useful



Representations emerge from coordinating all three."

**[13:20 - 13:25] [A-CAM: Interviewer, asking for a concrete example.]**

**INTERVIEWER:**
"Show us a concrete example. How do these three coordinate to learn something specific?"

**[13:25 - 15:00] [B-CAM: LeCun, building the example step-by-step.]**

**LECUN:**
"Let's use learning to recognize cats. Here's how the three elements coordinate:

**Architecture Prior (CNN):**

- Assumes spatial locality: cats have local features (whiskers, ears, eyes)


- Assumes translation invariance: cat is cat anywhere in image


- Provides hierarchical structure: simple features → complex features



**Data (images of cats):**

- Provides examples of what cats look like


- Shows variation: different breeds, poses, lighting


- Contains structure: cats have consistent patterns (fur, face shape, etc.)



**Objective (classification):**

- Minimize error: correctly identify cats vs. not-cats


- Pressure to learn discriminative features


- Ignore irrelevant variations (background, lighting)"



**[14:15] [SCREEN: Split screen showing the three forces acting on the network during training, with representation space evolving.]**

**LECUN:**
"**Coordination in action:**

Early training:

- Layer 1 learns edges (useful for many tasks, shaped by architecture)


- Layer 2 learns textures (cat fur patterns, shaped by data + objective)


- Layer 3 learns cat-specific patterns (ears, whiskers, shaped by all three)


- Layer 4 learns 'catness'—abstract concept (emergent from coordination)



The representation of 'cat' wasn't programmed. It was coordinated into existence by architecture + data + objective working together."

### SEGMENT 7: Why Supervised Learning Is Limited (5 min)

**[15:00 - 15:05] [A-CAM: Interviewer, transitioning to the limitation.]**

**INTERVIEWER:**
"This coordination architecture—data, objective, architecture—has revolutionized AI. But you've been arguing that supervised learning hits a ceiling. Why?"

**[15:05 - 15:50] [B-CAM: LeCun, explaining the three limitations.]**

**LECUN:**
"Three fundamental problems:

**Problem 1: Labels are expensive.**

ImageNet: 1.2 million images, hand-labeled by humans. Years of effort, millions of dollars. You can't label the whole world—there's not enough human labor. This is a bottleneck that doesn't scale."

**[15:25] [SCREEN: Comparison showing labeled data (tiny fraction) vs. unlabeled data (vast majority).]**

**LECUN:**
"We're drowning in data—videos, images, text, sensor readings—but only a tiny fraction is labeled. Supervised learning can only use the labeled fraction. We're ignoring 99.9% of available information."

**[15:50 - 15:55] [A-CAM: Interviewer, asking about the second problem.]**

**INTERVIEWER:**
"What's the second limitation?"

**[15:55 - 16:40] [B-CAM: LeCun, explaining the shallowness of supervision.]**

**LECUN:**
"**Problem 2: Labels are limiting.**

When you train on ImageNet, the network only learns what the labels specify: 'this is a cat, this is a dog.' It doesn't learn:

- Texture and material properties


- 3D structure and geometry


- Physics and dynamics


- Behavior and interaction



All that rich information is in the images, but the label 'cat' doesn't capture it. The network learns a shallow representation optimized for classification, not a deep understanding of the world."

**[16:40 - 16:45] [A-CAM: Interviewer, asking about generalization.]**

**INTERVIEWER:**
"And the third problem?"

**[16:45 - 17:30] [B-CAM: LeCun, explaining the generalization failure.]**

**LECUN:**
"**Problem 3: Labels are task-specific.**

Train on ImageNet → good at ImageNet classification. New task → need new labels. The representations don't transfer broadly. You're learning task-specific features, not general understanding."

**[17:05] [SCREEN: Diagram showing supervised learning locked to specific tasks vs. self-supervised learning enabling broad transfer.]**

**LECUN:**
"This is why I've been pushing self-supervised learning. It's not just more efficient—it's fundamentally different. It learns richer representations from the structure of data itself."

### SEGMENT 11: Beyond Feedforward—Coordination Loops (2 min)

**[25:50 - 25:55] [A-CAM: Interviewer, asking about the broader architecture.]**

**INTERVIEWER:**
"Current AI systems are fundamentally feedforward—input goes in, output comes out. You're saying the future is coordination loops. What's the difference?"

**[25:55 - 26:30] [B-CAM: LeCun, explaining the paradigm shift.]**

**LECUN:**
"**Current AI: Feedforward pipelines**
Input → Processing → Output

One-shot. No feedback. No iteration. No world model. No planning."

**[26:10] [SCREEN: Diagram showing simple feedforward pipeline.]**

**LECUN:**
"**Future AI: Coordination loops**
Perception ↔ World Model ↔ Action
↓ ↓ ↓
Continuous Coordination"

**[26:30 - 26:35] [WHITEBOARD CAM: LeCun draws coordination loop architecture.]**

**LECUN:**
"How it works:"

**[26:35 - 27:30] [B-CAM: LeCun, explaining the loop.]**

**LECUN:**
"**Perception:** Sense the environment—what's happening now

**World Model:** Predict what will happen—internal simulation

**Action:** Choose action based on prediction

**Feedback:** Observe outcome, compare to prediction

**Learning:** Update world model based on prediction error

**Iterate:** Continuous coordination"

**[27:00] [SCREEN: Animation showing the coordination loop in action—robot learning to manipulate objects through prediction and feedback.]**

**LECUN:**
"This is how biological intelligence works:

- Vision: Predict what you'll see, compare to actual (saccades, attention)


- Motor control: Predict consequences, adjust in real-time


- Cognition: Simulate scenarios, evaluate outcomes, choose actions



Not feedforward pipelines—coordination loops all the way down."

### SEGMENT 12: The Call to Action (3 min)

**[27:30 - 27:35] [A-CAM: Interviewer, bringing it to a close.]**

**INTERVIEWER:**
"We're in 2026. What needs to happen in AI research? What's your message to the field?"

**[27:35 - 28:50] [B-CAM: LeCun, delivering his vision with conviction.]**

**LECUN:**
"The AI field is at a crossroads. We've achieved incredible success with supervised learning—vision, language, games. But we're hitting the ceiling. To get to human-level intelligence, we need a paradigm shift.

**What must change:**

**Stop scaling supervised learning.** Bigger models trained on more labeled data won't get us to AGI. We need different architectures.

**Build coordination architectures.** Systems that learn from observation and interaction, like babies. Self-supervised learning, world models, coordination loops.

**Learn from biology.** The brain is our blueprint. Coordinated subsystems, prediction-based learning, continuous feedback loops.

**Focus on representations, not tasks.** Don't optimize for narrow benchmarks. Build systems that learn rich representations that transfer broadly."

**LECUN:**
"Intelligence emerges from coordination. Always has, always will. From CNNs to transformers to JEPA, the pattern is clear: coordinate multiple information streams, and intelligence emerges. Don't just scale—coordinate. Don't just supervise—let the system learn from the structure of the world. This is the architecture of learning. This is the path to AGI. And I've been right before when everyone said I was wrong."
